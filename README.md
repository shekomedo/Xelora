# برومبت تنفيذي: تطبيق أندرويد لتنظيف وتحليل البيانات وإدارة شيتات الإكسل

انسخ النص التالي كاملاً وأرسله مباشرة لأداة ذكاء اصطناعي قادرة على تنفيذ الأكواد فعلياً (مثل **Claude Code**، أو أي بيئة تطوير متصلة بذكاء اصطناعي). هذا البرومبت مكتوب ليبدأ التنفيذ الفعلي فوراً دون توقف لطلب تأكيد إضافي، إلا عند الضرورة القصوى.

---

## البرومبت

أنت خبير مزدوج التخصص:
1. **خبير تطوير تطبيقات أندرويد** محترف (Kotlin/Jetpack Compose أو Flutter، اختر الأنسب تقنياً وابدأ به فوراً)، لديك خبرة عميقة في بناء تطبيقات إنتاجية احترافية بواجهات مستخدم (UI/UX) عصرية وسهلة الاستخدام.
2. **خبير تحليل بيانات وهندسة بيانات (Data Engineer / Data Analyst)** متخصص في التعامل مع ملفات Excel وWord، **تنظيف البيانات (Data Cleaning) وتحليلها (Data Analysis) معاً**، إعادة الهيكلة، والتحليل الإحصائي والمالي، وعلى دراية بأحدث المكتبات والأدوات في هذا المجال (مثل Apache POI، OpenXML SDK، وأحدث تنسيقات ملفات xlsx/docx).

### المطلوب منك (نفّذ فوراً، لا تكتفِ بالشرح):
ابدأ الآن ببناء تطبيق أندرويد احترافي متكامل يعمل كـ"أداة ذكية لتنظيف وتحليل البيانات وشيتات الإكسل" حسب المواصفات التالية. لا تطلب موافقة قبل البدء — اتخذ أفضل القرارات التقنية بنفسك، ونفّذها مباشرة خطوة بخطوة (إعداد المشروع، ثم الشاشات، ثم منطق العمل، ثم الربط)، واعرض تقدمك أولاً بأول.

### 1. الوظائف الأساسية (Core Features)
- **تنظيف البيانات تلقائياً**: إزالة التكرارات، تصحيح التنسيقات، حذف الخلايا الفارغة أو الأخطاء، توحيد صيغ التاريخ والأرقام والنصوص.
- **تحليل البيانات**: استخراج رؤى وتقارير تلقائية من البيانات (اتجاهات، أنماط، قيم شاذة، ملخصات إحصائية) بجانب التنظيف — الاثنان معاً جزء أساسي من التطبيق وليس التنظيف فقط.
- **إعادة ترتيب الشيتات أبجدياً**: ترتيب أسماء الشيتات (Sheets) وأيضاً ترتيب الصفوف/الأعمدة داخل كل شيت أبجدياً حسب اختيار المستخدم.
- **الترتيب حسب الفترة الزمنية**: تنظيم البيانات حسب التاريخ (يومي/أسبوعي/شهري/سنوي) مع إمكانية تصفية فترة محددة.
- **حساب تكلفة المنتج**: حاسبة تكلفة تلقائية تأخذ بعين الاعتبار (تكلفة الخامات + العمالة + المصاريف التشغيلية + هامش الربح) وتُخرج تقرير تكلفة وربح لكل منتج.
- **إحصائيات وتحليل الأرباح**: رسوم بيانية ومؤشرات (KPIs) توضح الأرباح، أعلى المنتجات مبيعاً، تحليل الاتجاهات (Trends).

### 2. الاستيراد والتصدير (Import/Export)
- استيراد ملفات **Word (.docx)** و **Excel (.xlsx/.xls/.csv)** لتطبيق أدوات التنظيف والتحليل عليها مباشرة.
- تصدير النتائج كملفات **Excel جاهزة ومنسقة احترافياً** (بألوان، عناوين، جداول محورية إن أمكن) بنقرة واحدة.

### 3. صفحة الإعدادات (Settings)
- خيارات تخصيص (اللغة، الثيم الداكن/الفاتح، صيغة التاريخ الافتراضية، العملة المستخدمة في حساب التكلفة، نسخ احتياطي تلقائي).

### 4. المتطلبات التقنية
- استخدام أحدث المكتبات والأدوات المتاحة لمعالجة ملفات Excel وWord على أندرويد.
- كود نظيف، منظم بمعمارية واضحة (MVVM أو Clean Architecture)، قابل للصيانة والتوسعة.
- دقة عالية جداً في العمليات الحسابية والإحصائية (لا هامش خطأ في حسابات التكلفة أو الترتيب).
- أداء سريع حتى مع ملفات إكسل كبيرة الحجم.

### 5. واجهة المستخدم (UI/UX)
- تصميم احترافي، بسيط، وسهل الاستخدام حتى لغير المتخصصين.
- **شاشة Splash Screen متطورة** بتصميم خيالي/مستقبلي (Fantasy/Futuristic) مبهر بصرياً (تأثيرات حركية، إضاءة، انتقالات ناعمة) تعكس احترافية التطبيق.

### طريقة التنفيذ (إلزامية):
1. **لا تكتفِ بالشرح النظري** — أنشئ فعلياً هيكل المشروع الكامل (ملفات ومجلدات الكود) وابدأ بكتابة الأكواد الحقيقية مباشرة.
2. اختر بنفسك المعمارية التقنية الأنسب (Framework + المكتبات) واذكرها في سطر واحد فقط قبل الشروع في العمل، دون انتظار رد مني.
3. ابنِ التطبيق تدريجياً وبشكل عملي:
   - إعداد المشروع (Project Setup) وهيكلة الملفات.
   - شاشة Splash Screen الاحترافية أولاً.
   - الشاشة الرئيسية وشاشات الاستيراد/التصدير.
   - منطق تنظيف البيانات والترتيب الأبجدي والزمني.
   - منطق تحليل البيانات (إحصائيات، رؤى، اتجاهات) كوحدة مستقلة إلى جانب التنظيف.
   - منطق حساب التكلفة والتحليل المالي/الإحصائي للأرباح.
   - صفحة الإعدادات.
   - شاشة "حول التطبيق" لعرض حقوق الملكية.
4. بعد كل مرحلة، اعرض ملخصاً قصيراً لما تم إنجازه، ثم انتقل مباشرة للمرحلة التالية دون توقف لطلب إذن، إلا إذا واجهت قراراً حرجاً يتطلب توضيحاً مني (مثال: اسم حزمة التطبيق النهائي، أو صلاحيات خاصة).
5. تأكد من أن الكود قابل للتشغيل والاختبار فعلياً وليس مجرد أمثلة توضيحية.
6. اختم بملف أو قسم يلخص طريقة تشغيل المشروع (كيفية البناء والتجربة).
